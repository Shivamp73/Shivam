HOW I WORK

Frame. Build. Ship.

The same three moves on every project, whether it's a two-week sprint or a six-month system.

FRAME

Understand the real constraint

Talk to the people who'll use it, map the workflow as it actually happens, and name the one problem worth solving first.

BUILD

Design in the medium it ships in

Prototype in code where it matters, so what gets tested is close to what gets shipped, and feedback lands on the real thing.

SHIP

Hand off something engineers want

Specs, states, edge cases and a working component library — not a static file that leaves the details to guesswork.
